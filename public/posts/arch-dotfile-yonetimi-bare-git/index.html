Arch Linux’ta Dotfile Yönetimi: Bare Git Repo ile Sıfır Sembolik Link

2026-04-28Muhammet Fatih Şahan2 dk okuma (402 kelime)
#linux  #arch-linux  #git  #dotfiles  #terminal  #automation  #ricing 

Her Linux kullanıcısının er ya da geç karşılaştığı o an gelir: Yeni bir makine kuruyorsunuz ve tüm o özenle düzenlediğiniz yapılandırmaları sıfırdan kurmak zorunda kalıyorsunuz. Ya da stow ile sembolik link yönetimi yaparken kendinizi bir karmaşanın içinde buluyorsunuz.

Bugün size hem daha temiz, hem daha güçlü bir çözüm sunacağım: Bare Git Repository.

Problem: Neden Normal Git Repo Kullanmıyoruz?#

Home dizininizi (~/) doğrudan bir git reposuna çevirmek teoride mümkün, ancak pratikte felakettir. git status çalıştırdığınızda tüm home dizinindeki takip edilmeyen dosyaları görürsünüz — binlerce satır çıktı sizi karşılar.

Peki ~/.config dizinini ayrı bir repo olarak mı tutacaksınız? Bu sefer de senkronizasyon sorunu çıkıyor.

Çözüm: Bare Repository Yaklaşımı#

Bare bir repository, çalışma ağacı (working tree) olmayan özel bir git reposudur. Normalde sunucularda kullanılan bu yapıyı, dotfile yönetimi için kendi lehimize kullanacağız.

1. Adım: Repo’yu Başlatın# 

git init --bare $HOME/.dotfiles

Bu komut ~/.dotfiles dizininde git’in iç yapısını oluşturur ama hiçbir dosyayı dokunmaz.

2. Adım: Alias’ı Tanımlayın# 

alias dotfiles='git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME'

Bu alias’ı kalıcı hale getirmek için ~/.bashrc veya ~/.config/fish/config.fish dosyanıza ekleyin. Artık dotfiles komutunu, home dizinini çalışma ağacı olarak kullanan özel bir git komutu gibi kullanabilirsiniz.

3. Adım: Takip Edilmeyenleri Gizleyin# 

dotfiles config --local status.showUntrackedFiles no

Bu ayar sayesinde dotfiles status çalıştırdığınızda sadece takip ettiğiniz dosyaları göreceksiniz.

Günlük Kullanım#

Artık dotfile yönetimi gerçek anlamda git iş akışı haline geldi:

# Yeni bir config dosyasını takibe alın
dotfiles add ~/.config/fish/config.fish

# Değişikliği commit'leyin
dotfiles commit -m "feat: fish shell update alias güncellendi"

# GitHub'a push edin
dotfiles push origin main

# Durumu kontrol edin
dotfiles status

Neden Harika?#

Özellik Stow (Sembolik Link) Bare Git
Sembolik link karmaşası ❌ Var ✅ Yok
Gerçek dosya yolu korunur ❌ Hayır ✅ Evet
Git geçmişi ⚠️ Dolaylı ✅ Doğrudan
Yeni makineye kurulum Karmaşık Tek komut
Araç bağımlılığı stow gerekli Sadece git

Yeni Makineye Kurulum#

Tüm bunun güzelliği şu komutta saklı:

# GitHub'dan clone et
git clone --bare https://github.com/fatihnorthman/dotfiles $HOME/.dotfiles

# Alias'ı geçici olarak tanımla
alias dotfiles='git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME'

# Dosyaları yerleştir
dotfiles checkout

# Takip edilmeyenleri gizle
dotfiles config --local status.showUntrackedFiles no

Beş dakikada yeni makinenizde tüm yapılandırmalarınız hazır.

Sonuç#

Bare git repository yöntemi, dotfile yönetiminin en “Linux way” çözümüdür: Ek araç yok, sembolik link karmaşası yok, sadece git. Bir kez kurduğunuzda, her yeni makinede beş dakikada tam yapılandırılmış bir sistem sizi bekliyor olacak.

Kural: Bir işi doğru yapmak için önce doğru aracı seçin.
