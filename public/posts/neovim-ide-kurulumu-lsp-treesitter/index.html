Neovim’i IDE’ye Dönüştürmek: LSP, Treesitter ve Lazy.nvim ile Sıfırdan Kurulum

2026-04-25Muhammet Fatih Şahan2 dk okuma (387 kelime)
#linux  #neovim  #lua  #lsp  #terminal  #ricing  #development 

VS Code’dan Neovim’e geçiş kararı aldığımda önümde iki seçenek vardı: Hazır bir dağıtım kullanmak (NvChad, LazyVim) ya da her şeyi sıfırdan inşa etmek. İkincisini seçtim. Çünkü kullandığınız aracın her satırını anlamak, sizi daha iyi bir sistem mimarı yapar.

Bu yazıda, minimal ama profesyonel bir Neovim kurulumunu nasıl oluşturduğumu paylaşıyorum.

Neden Neovim?#

Kriter VS Code Neovim
RAM Kullanımı ~400-800 MB ~20-50 MB
Başlangıç Süresi 2-5 saniye <50ms
Özelleştirme Sınırlı Sonsuz (Lua)
Klavye Odaklı Kısmen Tamamen
SSH Üzerinden ❌ ✅ Mükemmel

Bir altyapı mimarı olarak sunucular üzerinde çalışmanın büyük kısmı SSH bağlantıları üzerinden geçer. Neovim burada rakipsiz.

Yapı: Lazy.nvim ile Eklenti Yönetimi#

Modern Neovim yapılandırmaları için standart dizin yapısı şöyle görünür:

~/.config/nvim/
├── init.lua              # Giriş noktası
└── lua/
    ├── config/
    │   ├── options.lua   # Temel ayarlar
    │   ├── keymaps.lua   # Kısayollar
    │   └── lazy.lua      # Eklenti yöneticisi
    └── plugins/
        ├── lsp.lua       # Dil sunucusu
        ├── treesitter.lua
        ├── telescope.lua
        └── ui.lua

init.lua — Giriş Noktası# 

-- Lua modüllerini yükle
require("config.options")
require("config.keymaps")
require("config.lazy")

LSP: Zekâyı Editöre Taşımak#

Language Server Protocol, editörden bağımsız otomatik tamamlama, hata gösterimi ve “tanıma git” özelliklerini sağlar.

-- lua/plugins/lsp.lua
return {
  "neovim/nvim-lspconfig",
  dependencies = {
    "williamboman/mason.nvim",
    "williamboman/mason-lspconfig.nvim",
    "hrsh7th/nvim-cmp",
  },
  config = function()
    require("mason").setup()
    require("mason-lspconfig").setup({
      ensure_installed = { "lua_ls", "bashls", "yamlls", "dockerls" },
      automatic_installation = true,
    })

    local lspconfig = require("lspconfig")
    local capabilities = require("cmp_nvim_lsp").default_capabilities()

    -- Her dil sunucusu için ortak ayarlar
    local on_attach = function(_, bufnr)
      local map = function(keys, func, desc)
        vim.keymap.set("n", keys, func, { buffer = bufnr, desc = desc })
      end
      map("gd", vim.lsp.buf.definition, "Tanıma Git")
      map("K",  vim.lsp.buf.hover,      "Belgeyi Göster")
      map("<leader>rn", vim.lsp.buf.rename, "Yeniden Adlandır")
    end

    lspconfig.lua_ls.setup({ capabilities = capabilities, on_attach = on_attach })
    lspconfig.bashls.setup({ capabilities = capabilities, on_attach = on_attach })
  end,
}

Treesitter: Sözdizimi Anlayışı#

Treesitter, regex tabanlı syntax highlighting’in aksine kodu gerçekten parse eder. Sonuç: Mükemmel renklendirme ve kod yapısı anlayışı.

-- lua/plugins/treesitter.lua
return {
  "nvim-treesitter/nvim-treesitter",
  build = ":TSUpdate",
  config = function()
    require("nvim-treesitter.configs").setup({
      ensure_installed = { "lua", "bash", "python", "yaml", "toml", "markdown" },
      highlight = { enable = true },
      indent   = { enable = true },
    })
  end,
}

Sonuç: Araç Sahibi Olmak#

Bugün öğrendiğiniz her Neovim konfigürasyonu satırı, size ait. NvChad kullandığınızda başkasının kararlarını kullanırsınız. Sıfırdan kurduğunuzda ise her kısayol, her renk, her davranış sizin tercihlerinizin yansımasıdır.

Sunucu üzerinde ssh fatih@server nvim important-config.yaml yazdığınızda ve tam gücüyle çalışan editörünüzü gördüğünüzde, bu farkı anlarsınız.

Hatırlatma: Araçları anlayarak kullanın, körü körüne değil.
